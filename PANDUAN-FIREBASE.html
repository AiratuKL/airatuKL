Panduan Firebase AiratuKL

Ikuti langkah bergambar ini dari atas ke bawah agar pendaftaran email, login Google, OTP telepon, database, dan upload foto dapat digunakan.

⚠️ Penting: masuk dengan akun Google pemilik proyek, lalu buka console.firebase.google.com dan pilih proyek airatukl. Tampilan Firebase dapat sedikit berubah, tetapi nama menu tetap serupa.
1

Buka Authentication

Firebase Console → proyek airatukl → Build → Authentication
  1. Di sidebar kiri, buka bagian Build.
  2. Klik Authentication.
  3. Jika muncul tombol Get started / Mulai, klik tombol tersebut.

Aktifkan Phone/OTP

Authentication → Sign-in method → Add new provider → Phone
  1. Pilih provider Phone.
  2. Aktifkan lalu klik Save.
  3. Untuk pengujian tanpa mengirim SMS, tambahkan nomor uji di bagian Phone numbers for testing, misalnya +6281234567890 dan kode 123456.

Pengiriman SMS nyata dapat meminta konfigurasi billing/kuota sesuai kebijakan Firebase dan negara tujuan. Nomor uji adalah cara termudah untuk tes awal.

5

Tambahkan Authorized Domain

Authentication → Settings → Authorized domains
  1. Buka tab Settings.
  2. Cari bagian Authorized domains.
  3. Pastikan localhost ada. Jika belum, klik Add domain dan masukkan localhost tanpa http:// dan tanpa port.
  4. Saat sudah hosting, tambahkan domain asli, misalnya chat.domainanda.com.

Buat Firestore Database

Build → Firestore Database → Create database
  1. Klik Firestore Database di sidebar.
  2. Klik Create database.
  3. Pilih Start in production mode.
  4. Pilih lokasi terdekat dan konsisten. Lokasi tidak mudah diubah.
  5. Klik Enable, kemudian pasang Security Rules dari README.md.

Aktifkan Firebase Storage

Build → Storage → Get started
  1. Klik Storage di sidebar.
  2. Klik Get started.
  3. Pilih mode produksi dan lokasi, lalu selesaikan wizard.
  4. Buka tab Rules dan pasang Storage Rules dari README.md.

Jika Firebase meminta upgrade billing saat mengaktifkan Storage, ikuti instruksi paket pada Console. Ini berasal dari kebijakan proyek Firebase, bukan dari kode AiratuKL.

8

Jalankan AiratuKL dengan benar

  1. Ekstrak airatuKL11.zip.
  2. Pastikan Node.js 18 atau lebih baru sudah terpasang.
  3. Buka Terminal/Command Prompt di dalam folder chatconnect.
  4. Jalankan perintah berikut satu per satu:
# masuk ke folder proyek (sesuaikan lokasinya)
cd chatconnect

# unduh dependency
npm install

# mulai server
npm start

Setelah terminal menampilkan AiratuKL aktif, buka Chrome dan ketik:

http://localhost:3000

Jangan klik dua kali file login.html. Alamat yang diawali file:/// tidak dapat digunakan untuk Google OAuth.

9

Uji sampai berhasil

  • ☐ Buka http://localhost:3000/register.html.
  • ☐ Klik Daftar dengan Google; pemilih akun Google harus muncul.
  • ☐ Coba daftar menggunakan email yang belum pernah dipakai.
  • ☐ Periksa Firebase → Authentication → Users; akun harus muncul.
  • ☐ Periksa Firestore → Data → koleksi users; profil harus muncul.
  • ☐ Coba keluar lalu login kembali.
Jika muncul “unauthorized-domain”: ulangi Langkah 5. Jika popup tidak muncul, izinkan popup browser. Jika npm tidak dikenali, instal Node.js LTS dari nodejs.org, lalu buka ulang terminal.
